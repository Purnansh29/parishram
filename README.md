# Parishram | Project Phases Roadmap

Welcome to the **Parishram School Management System** repository.

This project is currently executing the following upcoming phases to build a unified full-stack MERN application. 

## Upcoming Project Phases

### ⏳ Phase 1: Authentication & Core Architecture
- Implementation of JWT-based secure login protocols.
- Setup of Role-Based Access Controls (Super Admin, Admin, Teacher, Student, Parent).
- Database normalization mapping for Mongoose schemas (Users, Roles, Sessions).
- Initializing the Node.js / Express backend routing module.

### ⏳ Phase 2: Student & Staff Management
- Complete CRUD APIs and admin UI for Student admissions & profiles.
- Staff allocation mapping, including department splits and teacher onboarding features.
- Dynamic digital ID card generation pipeline.

### ⏳ Phase 3: Academic & Timetable Management
- Building interactive UI components to generate, render, and manage class schedules and timetables.
- Establishing daily attendance registers (Biometric/Manual sync).
- Syllabus and daily homework payload tracking systems.

### ⏳ Phase 4: Financial Module integration
- Mapping out standard fee structures per grade/class.
- Invoicing systems, fine-calculations, and payment records.
- Exporting financial records into comprehensive spreadsheet reports.

### ⏳ Phase 5: Examinations & Reporting
- Configuration interfaces for school terms, tests, and exam marks.
- Automated report card processing algorithms based on grading bounds.
- Dedicated parent portals to analyze student progression charts.

### ⏳ Phase 6: Communication Hub 
- Global and segment-targeted announcement boards (Events/Holidays).
- Email and SMS trigger integrations (Twilio/Nodemailer) for instant alerts regarding attendance or fees.

### ⏳ Phase 7: Final Polish & Master Deployment
- Full E2E testing using Cypress for the React frontend and Jest for the Express API.
- Cloud deployment pipelines (AWS/Vercel) and CI/CD GitHub workflows.
- Performance optimization and rigorous frontend caching configurations.
